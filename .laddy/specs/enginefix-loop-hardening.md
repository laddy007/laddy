# enginefix-loop-hardening — make the autonomous dev-loop robust on the VPS

## Goal
Harden the dev-loop against three failure modes found while dogfooding the
smoke1 run on the VPS, plus a doc clarification. A tested baseline
implementation is ALREADY PRESENT on this branch — treat it as a starting
point: review it against the acceptance criteria, keep it if it holds, and
refine ONLY if you can make it genuinely better without regressing the suite.

## Root-cause context (why these changes)
1. env.vps generation loses TEST_COMMANDS. `scripts/vps-onboard.sh` writes the
   generated env.vps ending at `NTFY_TOPIC=<val>` with NO trailing newline
   (command substitution strips it). A later hand-append (e.g. TEST_COMMANDS)
   glues onto the NTFY_TOPIC line; bash parses it as one assignment and
   TEST_COMMANDS is silently never set → the fast gate falls back to a default
   that activates a `.venv` the VPS never created → every round fails
   `.venv/bin/activate: No such file or directory`.
2. Stale run-lock blocks all re-runs. `orchestrator/queue.py` `run_lock` uses
   O_EXCL only and never checks pid liveness, so a lock left by a crashed loop
   refuses every later run (rc=4) until a human deletes it — killing autonomy
   after any crash.
3. Detached loop log looks empty / dead. `orchestrator/loop.py` prints nothing
   to stdout (progress goes to the per-task iteration-log); `orchestrator/run.py`
   prints only the final terminal state; and `scripts/kickoff.sh` detaches
   `nohup python3` WITHOUT `-u`, so a crash before that final print swallows the
   block-buffered output → an empty $LOG that reads like an instant death.

## Scope
In: `scripts/vps-onboard.sh`, `orchestrator/queue.py`, `orchestrator/artifacts.py`,
`scripts/kickoff.sh`, `env.vps.example`, and their tests under `tests/`.
Out: any behavioural change to the convergence/gate/merge policy beyond what the
criteria below require; no new dependencies.

## Acceptance criteria
1. env.vps trailing newline: the env.vps generated by `scripts/vps-onboard.sh`
   ends with a newline, so a later appended `VAR=value` line parses as its own
   independent variable (NTFY_TOPIC and the appended var both resolve). Test.
2. Stale-lock auto-reclaim: `run_lock` reclaims a lock whose recorded pid is no
   longer alive, and still refuses a lock held by a live process. Tests for both
   the dead-pid and live-holder cases + a pid-liveness unit test. The existing
   per-task exclusivity contract (live holder → QueueLocked) must still pass.
3. Observable detached run: `scripts/kickoff.sh` launches the loop unbuffered
   (`-u`/PYTHONUNBUFFERED) and each iteration-log entry is mirrored to the
   detached $LOG as it happens (env-gated so default CLI/test runs stay quiet).
   Test for the gated heartbeat.
4. Docs: `env.vps.example` explains the fast inner gate vs the authoritative
   Docker gate, ships a TEST_COMMANDS example that bootstraps the venv (the VPS
   loop has no separate setup step), and drops the duplicate-variable template
   trap.
5. No regression: the full existing suite stays green; every new code path is
   covered so diff-coverage holds.
